## Monitor Resolution

Workspace 1 window positions are **pixel-exact for 1920×1200**. If your monitor is a different resolution, update the `move=` and `size=` values in these windowrules in `dotfiles/home/.config/hypr/hyprland.conf`:

### Workspace 1 Layout (1920×1200 reference)

```
                          ┌────────────────────────┐
                          │  kitty-ws1-top         │
                          │  move=984 80           │
                          │  size=888×512          │
                          │  → runs [AI_ASSISTANT]         │
                          ├────────────────────────┤
                          │  kitty-ws1-bottom      │
                          │  move=984 640          │
                          │  size=888×512          │
                          └────────────────────────┘
```

Windowrule blocks to update (search for `name =` in `hyprland.conf`):

| Windowrule name | `move=` | `size=` |
|---|---|---|
| `kitty-ws1-top` | `984 80` | `888 512` |
| `kitty-ws1-bottom` | `984 640` | `888 512` |
| `kitty-ws2-nvim` | `48 48` | `1824 1104` |

The monitor line in `hyprland.conf` uses `preferred` which auto-detects resolution:
```
monitor=,preferred,auto,1.0
```
This is fine to leave as-is — only the windowrule positions need manual adjustment.

## Username

Two files hardcode the `admin` username and must be updated if your username differs:

### 1. `ansible/configurations/roles/core_packages/tasks/install.yml`

The `yay` build path:
```yaml
path: /home/admin/yay/
```
Change `admin` to your username.

### 2. `dotfiles/home/.config/nvim/lua/config/autocmds.lua`

The project directory check that triggers terminal splits in Neovim:
```lua
local PROJECT_DIR = "/home/admin/andusystems-arch"
```
Change `admin` to your username.

### 3. `dotfiles/home/.config/hypr/hyprland.conf`

The workspace 1 and 2 autostart commands:
```
exec-once = [workspace 1 silent] kitty --class kitty-ws1-top --directory /home/admin/andusystems-arch -- [AI_ASSISTANT] --dangerously-skip-permissions
exec-once = [workspace 1 silent] kitty --class kitty-ws1-bottom --directory /home/admin/andusystems-arch
exec-once = [workspace 2 silent] kitty --class kitty-ws2-nvim --directory /home/admin/andusystems-arch -- nvim /home/admin/andusystems-arch
```
Change all four `/home/admin/` references to your home directory.

## Nightlight Location

Edit `dotfiles/home/.local/bin/nightlight.sh` and update the latitude/longitude:

```bash
wlsunset -l 40.0 -L -75.0 -t 3800 -T 6500
```

- `-l` = latitude (positive = North, negative = South)
- `-L` = longitude (positive = East, negative = West)

To find your coordinates: search "[your city] latitude longitude" or use `curl ipinfo.io`.

## Wallpapers

Copy images to `~/Wallpapers/`. Supported formats: `jpg`, `jpeg`, `png`, `webp`.

The `theme-switch.sh` script:
1. Picks a random wallpaper from `~/Wallpapers/` (avoids repeating the last one)
2. Sets it via `awww` with a `wipe` transition
3. Runs `matugen image <wallpaper>` to regenerate color templates
4. Reloads Waybar, Rofi, Kitty, Neovim, and swaync

To trigger manually at any time:
```bash
~/.local/bin/theme-switch.sh
```

## Adding an App to the Matugen Theming Pipeline

To have a new app's colors generated by matugen on each wallpaper change:

1. **Create a template** at `dotfiles/home/.config/matugen/templates/<app>.css` (or whatever format the app needs). Use `{{colors.<color_name>.default.hex}}` placeholders — see existing templates for reference.

2. **Register the template** in `dotfiles/home/.config/matugen/config.toml`:
   ```toml
   [templates.<app>]
   input_path = '~/.config/matugen/templates/<app>.css'
   output_path = '~/.config/<app>/colors.css'
   ```

3. **Add a reload step** in `dotfiles/home/.local/bin/theme-switch.sh` so the app picks up the new colors after each wallpaper change (e.g., `pkill -HUP -f <app>` or `pkill -USR2`).

## Fonts

The primary font is `Hack Nerd Font`. To change it, update these files:

| File | Setting |
|---|---|
| `dotfiles/home/.config/kitty/kitty.conf` | `font_family Hack Nerd Font Mono` and `font_size 11.0` |
| `dotfiles/home/.config/waybar/style.css` | `font-family: "Hack Nerd Font";` |
| `dotfiles/home/.config/rofi/config.rasi` | `font: "Hack Nerd Font 12";` |
| `dotfiles/home/.config/matugen/templates/kitty.conf` | `font_family` line |

Installed alternatives (already available): `JetBrains Mono Nerd Font`, `CascadiaCode Nerd Font`, `FiraCode Nerd Font`.

## Excluding a Config from Dotfile Tracking

To prevent a `~/.config/<app>/` directory from being adopted into the repo:

1. Add to `.gitignore`:
   ```
   dotfiles/home/.config/<app>/
   ```

2. Add to `ansible/configurations/roles/dotfiles/defaults/main.yml`:
   ```yaml
   dotfiles_exclude:
     - <app>
   ```
